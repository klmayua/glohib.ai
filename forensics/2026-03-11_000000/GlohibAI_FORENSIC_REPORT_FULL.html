Architecture Anti-Patterns

  1. Shared Database: All services share single PostgreSQL instance (no database-per-service)
  2. No API Gateway: Frontend directly calls individual services
  3. Tight Coupling: Services depend on each other's health
  4. No Circuit Breaker: Cascading failures possible
  5. No Event Bus: RabbitMQ configured but not used
